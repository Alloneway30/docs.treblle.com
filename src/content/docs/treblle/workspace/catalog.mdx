Publishing your API to the Catalog is a way to bring API Discoverability to technical *and* non-technical teams across your company. Any API Owner can go into API Settings and Publish the API, even if the API is marked Private (\*APIs are Private by default).

![](https://lh7-rt.googleusercontent.com/docsz/AD_4nXe3veWH0UIJmj5CWAEwV0B42OlXEY8GMD3hHtAjRHOeibs4tiexfbFCxq00oR4rR3KHaLK_Ux2yUx1f-gkatNxQXW5Jg1qOIAHnax7OcyC5tOsKriRi3xlglMrDx9IZgC6fdZY8AAwI3IFwyefo_OqoE0l7?key=MSO7VA8yHJQKt7Qqriwl4w)This will then open a modal, where the API Owner can add more details about the API for Catalog users to see.

![](https://lh7-rt.googleusercontent.com/docsz/AD_4nXep41pU6e5Cmz4Hpl51Eejbl10ZYvdDq7mIW1zjThyFDCBGYfnuDLUhXFNURqD5OlxlX2kzImyQlwIjzSw1JqlPn9jEa7hUSa3EfoEPIzDp3GGwOsNENNz2_zdOvoFOkEqHQS3zs-_mwpgCISlySblVRxuc?key=MSO7VA8yHJQKt7Qqriwl4w)**Category** - this describes the type of API from a high level

**Tags** - this helps further describe the API and allows you to use any internal keywords, names, or team references for the API

**Description** - this allows you to write a detailed description about what this API is, who it is for, and how it can be used

Once the API has been Published, it will appear on the Catalog where users can search, sort, and follow your company APIs. See Finding an API.
